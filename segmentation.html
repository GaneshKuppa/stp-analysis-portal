2 Hierarchical Clustering Implementation

Hierarchical clustering builds a tree of clusters (dendrogram) by iteratively merging the closest pairs of observations or clusters.

hierarchical_clustering.R
# Load and prepare segmentation data
seg.df <- read.csv("Data_Segmentation.csv", stringsAsFactors = TRUE)

# Recode binary factors to numeric (0/1)
seg.df$gender <- ifelse(seg.df$gender == "Male", 0, 1)
seg.df$ownHome <- ifelse(seg.df$ownHome == "ownNo", 0, 1)
seg.df$subscribe <- ifelse(seg.df$subscribe == "subNo", 0, 1)

# Standardize continuous variables ONLY
seg.df.sc <- seg.df
seg.df.sc[, c(1, 3, 4)] <- scale(seg.df[, c(1, 3, 4)])  # age, income, kids

# Compute Euclidean distance matrix
seg.dist <- dist(seg.df.sc, method = "euclidean")

# Perform hierarchical clustering (complete linkage)
seg.hc <- hclust(seg.dist, method = "complete")

# Cut tree to get 4 clusters
seg.hc.segment <- cutree(seg.hc, k = 4)

# Visualize dendrogram with cluster rectangles
plot(seg.hc, main = "Hierarchical Clustering Dendrogram", 
     xlab = "", sub = "", cex = 0.6)
rect.hclust(seg.hc, k = 4, border = "red")

# Visualize clusters in 2D space
library(cluster)
clusplot(seg.df, seg.hc.segment, 
        color = TRUE, shade = TRUE, labels = 4, lines = 0,
        main = "Customer Segments (Hierarchical)",
        xlab = "Component 1", ylab = "Component 2")

# Profile clusters (business interpretation)
aggregate(seg.df, list(Cluster = seg.hc.segment), mean)
Hierarchical Clustering Dendrogram
The dendrogram shows how observations merge into clusters. The height represents the distance at which clusters merge. Cutting the tree at different heights yields different numbers of clusters (k=2,3,4 shown).

3 K-Means Clustering Implementation

K-means partitions observations into k pre-specified clusters by minimizing within-cluster variance.

kmeans_clustering.R
# Determine optimal k using elbow method
set.seed(96743)  # For reproducibility
wss <- numeric(10)

for (i in 1:10) {
  km <- kmeans(seg.df.sc, centers = i, nstart = 25)
  wss[i] <- km$tot.withinss
}

# Plot elbow curve
plot(1:10, wss, type = "b", pch = 19,
     xlab = "Number of Clusters (k)", 
     ylab = "Within-Cluster Sum of Squares",
     main = "K-Means Elbow Method")
abline(v = 4, col = "red", lty = 2)  # Suggested k=4

# Run k-means with optimal k
seg.k <- kmeans(seg.df.sc, centers = 4, nstart = 25)

# Visualize clusters
clusplot(seg.df, seg.k$cluster,
        color = TRUE, shade = TRUE, labels = 4, lines = 0,
        main = "Customer Segments (K-Means)",
        xlab = "Component 1", ylab = "Component 2")

# Profile clusters
aggregate(seg.df, list(Cluster = seg.k$cluster), mean)

4 Model-Based Clustering (Mclust)

Mclust uses statistical models to identify clusters with different shapes, volumes, and orientations, automatically selecting the optimal number of clusters using Bayesian Information Criterion (BIC).

mclust_clustering.R
library(mclust)

# Run Mclust (automatically selects optimal k)
seg.mc <- Mclust(seg.df.sc)

# View model summary and optimal clusters
summary(seg.mc)
# Output shows: optimal clusters (G), model type, BIC value

# Visualize BIC values for different cluster solutions
plot(seg.mc, what = "BIC", main = "Mclust: BIC for Cluster Solutions")

# Visualize clusters
clusplot(seg.df, seg.mc$classification,
        color = TRUE, shade = TRUE, labels = seg.mc$G, lines = 0,
        main = paste("Model-Based Clustering (k =", seg.mc$G, ")",
        xlab = "Component 1", ylab = "Component 2")

# Profile clusters
aggregate(seg.df, list(Cluster = seg.mc$classification), mean)
Interpretation Framework

After clustering, always validate segments using:

  • ANOVA tests: Verify significant differences between clusters on key variables
  • Business relevance: Do segments align with strategic priorities?
  • Actionability: Can we reach and serve these segments effectively?
  • Stability: Do segments persist over time? (Validate with holdout sample)
